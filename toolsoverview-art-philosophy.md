# Command Constellation - Algorithmic Philosophy

## Concept: Knowledge Tree Visualization

A radial mind-map where the center represents "Claude Code" and branches extend outward to 14 domain clusters. Each domain is a colored node with its commands/skills as leaves. The visual metaphor is a tree of knowledge - roots at center, branches spreading to domains, leaves showing individual tools.

## Visual Design Principles

### 1. Radial Hierarchy
- **Center**: Core label "Claude Code Tools"
- **Ring 1**: 14 domain nodes arranged in a circle
- **Ring 2**: Individual commands branching from their domain

### 2. Color Coding by Domain
Each of the 14 domains gets a distinct, memorable color:
- UI & Frontend: Coral/Orange (#FF6B6B)
- Code Development: Electric Blue (#4ECDC4)
- Analysis & Review: Purple (#9B59B6)
- Debugging & Testing: Green (#2ECC71)
- Documentation: Gold (#F39C12)
- DevOps: Steel Blue (#3498DB)
- Git & Version: Crimson (#E74C3C)
- Planning: Teal (#1ABC9C)
- Parallel Execution: Magenta (#E91E63)
- AI & Automation: Cyan (#00BCD4)
- Session Management: Amber (#FFC107)
- Norwegian Grants: Nordic Blue (#5C6BC0)
- Plugin Development: Lime (#8BC34A)
- Misc/Hookify: Silver (#78909C)

### 3. Typography for Wall Reading
- Domain labels: 14-16pt bold
- Command names: 10-12pt
- Minimum contrast ratio for readability from 1.5m distance

### 4. Organic Branching
Curved bezier connections from center to domains, and from domains to commands. Creates a flowing, memorable tree structure rather than rigid grid.

## Interaction Model
- Seed changes reorganize branch angles slightly
- Parameters control:
  - Branch curvature
  - Node spacing
  - Color saturation
  - Animation speed (subtle pulse on nodes)

## A3 Landscape Constraints
- Canvas: 420mm x 297mm (at 96 DPI: ~1587 x 1123 pixels)
- Domains positioned to maximize space usage
- Commands flow outward in arcs, wrapping if needed
